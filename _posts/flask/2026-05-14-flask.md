在 Flask 开发中，**蓝图（Blueprint）** 是一个非常核心的概念。简单来说，它是为了**将一个大型项目拆分成多个小的、独立模块**而设计的工具。

当你的项目代码超过几百行，把所有的路由（URL）都写在主程序 `app.py` 里就会变成一场灾难。这时候，蓝图就是你的“救星”

```
login-form/
│
├── app.py                 # 主应用入口
├── auth/                  # 认证蓝图文件夹
│   ├── __init__.py        
│   └── views.py           # 登录、注册等视图逻辑
└── templates/             # 模板文件夹
    └── auth/
        └── login.html     # 登录页面 HTML
```

`auth/views.py`

```python
from flask import Blueprint, render_template, request, redirect, url_for, flash

# 创建名为 'auth' 的蓝图
# url_prefix 可以直接在这里指定，也可以在注册时指定
auth_bp = Blueprint('auth', __name__, url_prefix='/auth')

@auth_bp.route('/login', methods=['GET', 'POST'])
def login():
    if request.method == 'POST':
        # 获取表单提交的数据
        username = request.form.get('username')
        password = request.form.get('password')
        
        # 模拟数据库验证
        if username == 'admin' and password == '123456':
            flash('登录成功！', 'success')
            return redirect(url_for('index')) # 登录成功跳转到首页
        else:
            flash('用户名或密码错误！', 'danger')
            
    # GET 请求或登录失败时，渲染登录页面
    # 遵循规范，模板放在 templates/auth/ 目录下
    return render_template('auth/login.html')
```

`app.py`

```python
from flask import Flask
# 导入我们写好的蓝图对象
from auth.views import auth_bp

app = Flask(__name__)

# 注册蓝图
app.register_blueprint(auth_bp)

@app.route('/')
def index():
    return "<h1>这是系统首页（登录后可见）</h1>"

if __name__ == '__main__':
    app.run(debug=True)
```
