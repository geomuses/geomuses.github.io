# `ClientSession` 的生命周期管理

在 `aiohttp` 中，**绝对不要**为每一个请求创建一个新的 `session`。

`ClientSession` 内部维护着一个 **TCP 连接池**。复用 session 可以重用已有的连接（Keep-Alive），避免频繁的三次握手和四次挥手。

**正确做法：** 在 `main` 函数或 `class` 的初始化中开启，在程序结束时关闭。

```python
async def main():
    # 使用上下文管理器确保退出时清理连接
    async with aiohttp.ClientSession() as session:
        await fetch(session, url)
```

# 异步请求头的设置策略

请求头（Headers）可以全局设置，也可以针对单个请求设置。

- **全局设置**：适用于所有请求都需要携带的信息（如 `User-Agent`, `Accept-Language`）。
- **局部覆盖**：在具体的 `session.get()` 中传入，会覆盖全局设置。
```python
headers = {"User-Agent": "MyBot/1.0"}

async with aiohttp.ClientSession(headers=headers) as session:
    # 局部覆盖：为这个特定请求更换 Referer
    async with session.get(url, headers={"Referer": "https://google.com"}) as resp:
        pass
```

# 连接池与并发控制 (TCPConnector)

如果你不加限制地启动一万个协程，`aiohttp` 默认会尝试同时建立大量连接，这可能导致 `OSError: [Errno 24] Too many open files` 或被服务器拉黑。

我们需要通过 `TCPConnector` 来精细控制连接池：

```python
from aiohttp import TCPConnector

async def main():
    # limit: 总并发连接数限制（默认 100）
    # limit_per_host: 对同一个域名（IP）的最大并发限制（默认 0，即不限制）
    connector = TCPConnector(limit=50, limit_per_host=10, verify_ssl=False)
    
    async with aiohttp.ClientSession(connector=connector) as session:
        # 这里的请求将受到上述限制
        pass
```

# 代码

```python
import asyncio
import aiohttp

class WikiCrawler:
    def __init__(self):
        # 1. 设置连接池控制
        self.connector = aiohttp.TCPConnector(
            limit=20,            # 同时最多 20 个连接
            limit_per_host=5,     # 同一域名最多 5 个
            keepalive_timeout=30  # 连接保持时间
        )
        # 2. 设置全局请求头
        self.headers = {
            "User-Agent": "Mozilla/5.0 ...",
            "Accept": "text/html"
        }
        self.session = None

    async def __aenter__(self):
        self.session = aiohttp.ClientSession(
            connector=self.connector, 
            headers=self.headers
        )
        return self

    async def __aexit__(self, exc_type, exc_val, exc_tb):
        await self.session.close()

    async def fetch(self, url):
        async with self.session.get(url) as response:
            return await response.text()

async def main():
    urls = ["https://en.wikipedia.org/wiki/" + str(i) for i in range(10)]
    
    # 使用异步上下文管理器管理爬虫类
    async with WikiCrawler() as crawler:
        tasks = [crawler.fetch(url) for url in urls]
        results = await asyncio.gather(*tasks)
        print(f"完成了 {len(results)} 个页面的抓取")

if __name__ == "__main__":
    asyncio.run(main())
```

## 核心参数总结表

| **参数**               | **作用**      | **推荐建议**                                   |
| -------------------- | ----------- | ------------------------------------------ |
| **`limit`**          | 最大并发连接总数    | 根据内存和带宽设置，通常 50-100。                       |
| **`limit_per_host`** | 对单一域名的并发    | 极其重要。建议设为 5-10，防止被封。                       |
| **`verify_ssl`**     | 是否验证 SSL 证书 | 爬取某些老旧网站出错时可设为 `False`。                    |
| **`timeout`**        | 超时设置        | 必须设置！使用 `aiohttp.ClientTimeout(total=10)`。 |
