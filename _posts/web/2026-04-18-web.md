在异步编程中，`asyncio.wait_for()` 是处理“万一这玩意儿卡死或者太慢了怎么办”的必备工具。它就像给你的异步任务设了一个闹钟：**时间一到，还没干完，直接掐断并报错。**

#  关键机制：任务取消

有一点非常重要：**一旦发生超时，`wait_for` 会自动尝试取消（cancel）正在运行的任务。**

- 如果你的任务内部没有正确处理 `CancelledError`，它会立即停止。
    
- 如果你在任务里用了 `try...finally`，`finally` 块依然会执行，这方便你进行清理工作（比如关闭数据库连接）。
- 
- 建议直接捕获内置的 `TimeoutError`，而且你还可以使用更优雅的 `asyncio.timeout` 上下文管理器：

```python
import asyncio

async def eternal_task():
    print("任务开始，预计耗时 100 秒...")
    await asyncio.sleep(100)
    print("任务完成了！")
    
async def main():
    try:
        async with asyncio.timeout(2.0):
            await eternal_task()
    except TimeoutError:
        print("超时了，但在上下文管理器中看起来更整洁！")
```
