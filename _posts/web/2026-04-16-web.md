异步编程的核心在于**“非阻塞”**——当程序在等待（比如 IO、网络请求）时，它不会傻傻地等，而是去处理别的事情。

# 核心概念拆解

- **`async def` (定义)**：这不仅仅是一个普通函数，它定义了一个**协程函数 (Coroutine Function)**。当你调用它时，它不会立即执行代码，而是返回一个**协程对象**。
    
- **`await` (挂起)**：这是协程的“灵魂”。它告诉程序：“这里需要花点时间，你先去忙别的吧，等这边好了再回来叫我。”
    
- **事件循环 (Event Loop)**：它是协程的调度员。协程对象不能直接运行，必须被扔进事件循环中才能执行。

# 第一个异步 Hello World 实现

在 Python 3.7+ 中，我们通常使用 `asyncio.run()` 来启动最高层级的入口点。

Python

```python
import asyncio
import time

# 1. 使用 async def 定义协程函数
async def say_hello():
    print("Hello...")
    
    # 2. 使用 await 挂起执行
    # asyncio.sleep(1) 模拟一个耗时的 IO 操作（比如查数据库）
    await asyncio.sleep(1) 
    
    print("...World!")

# 3. 调用协程对象
if __name__ == "__main__":
    # 注意：直接调用 say_hello() 只会返回一个协程对象，不会打印任何东西
    # coroutine_object = say_hello() 
    
    # 我们需要使用 asyncio 的运行机制来驱动它
    asyncio.run(say_hello())
```
