实现一个异步资源管理器（Asynchronous Context Manager）需要遵循 Python 的 `async with` 协议。这要求类中必须实现两个特殊方法：`__aenter__` 和 `__aexit__`。

在异步编程中，这种模式非常强大，因为它能确保即使在发生异常的情况下，资源（如文件、数据库连接或网络套接字）也能被正确关闭。

 在 Python 的异步编程中，`async with` 语句是处理异步资源管理的核心工具。它基于 **异步上下文管理器 (Asynchronous Context Manager)** 协议，确保资源（如网络连接、数据库事务或文件句柄）即便在发生异常时也能被正确地初始化和关闭。

---

# 核心协议：`__aenter__` 与 `__aexit__`

要让一个对象支持 `async with`，它必须实现以下两个异步魔术方法：

- **`async def __aenter__(self)`**:
    
    - 在进入 `async with` 代码块之前被调用。
        
    - 用于执行异步初始化（如建立连接）。
        
    - 它的返回值会通过 `as` 关键字赋值给目标变量。
        
- **`async def __aexit__(self, exc_type, exc_val, exc_tb)`**:
    
    - 在离开代码块时被调用（无论是否发生异常）。
        
    - 用于执行异步清理（如释放锁、关闭连接）。
        
    - 如果返回 `True`，则会抑制代码块内抛出的异常。
        

手动实现协议

假设我们要模拟一个异步数据库连接器：

```python
import asyncio

class AsyncDatabase:
    def __init__(self, db_name):
        self.db_name = db_name

    async def __aenter__(self):
        print(f"正在异步连接数据库: {self.db_name}...")
        await asyncio.sleep(1)  # 模拟 IO 耗时
        self.conn = f"Connection_to_{self.db_name}"
        return self.conn  # 返回给 'as' 后的变量

    async def __aexit__(self, exc_type, exc_val, exc_tb):
        print(f"正在异步关闭数据库: {self.db_name}...")
        await asyncio.sleep(0.5)
        if exc_type:
            print(f"检测到异常: {exc_val}")
        # 返回 False（默认）让异常继续向上抛出

async def main():
    async with AsyncDatabase("db_name") as db:
        print(f"使用中: {db}")
        # 业务逻辑...

asyncio.run(main())
```

实际运用如下 

```python
import asyncio
from motor.motor_asyncio import AsyncIOMotorClient

async def insert_user():
    # 1. 初始化连接
    # motor 的客户端在内部管理连接池
    uri = "mongodb://localhost:27017"
    client = AsyncIOMotorClient(uri)
    
    # 2. 选择数据库和集合
    db = client.test_database
    collection = db.users

    # 准备数据
    user_data = {"name": "geo", "age": 30}

    try:
        # 3. 异步执行插入操作
        result = await collection.insert_one(user_data)
        print(f"数据插入成功，ID: {result.inserted_id}")
    except Exception as e:
        print(f"插入失败: {e}")
    finally:
        # 4. 关闭连接（在长连接应用中通常由 client 自己管理，但在脚本结束前建议手动关闭）
        client.close()
        print("连接已关闭")

# 启动异步任务
if __name__ == "__main__":
    asyncio.run(insert_user())
```

使用上下文管理器管理生命周期

```python
class MongoManager:
    def __init__(self, uri):
        self.uri = uri
        self.client = None

    async def __aenter__(self):
        # 异步初始化资源
        self.client = AsyncIOMotorClient(self.uri)
        print("MongoDB 客户端已启动")
        return self.client

    async def __aexit__(self, exc_type, exc_val, exc_tb):
        # 异步关闭资源
        if self.client:
            self.client.close()
            print("MongoDB 客户端已安全关闭")

async def main():
    async with MongoManager("mongodb://localhost:27017") as client:
        db = client.test_database
        user = {"name": "geo", "age": 30}
        await db.users.insert_one(user)

asyncio.run(main())
```

# 使用 `contextlib` 简化实现

手动写类有时略显繁琐。Python 提供了 `@asynccontextmanager` 装饰器，让你通过 **生成器 (Generator)** 的方式快速实现。

```python
from contextlib import asynccontextmanager

@asynccontextmanager
async def get_connection(host):
    # --- __aenter__ 部分 ---
    print(f"连接到 {host}")
    conn = {"host": host, "status": "connected"}
    try:
        yield conn  # yield 处暂停，将值传给 'as'
    finally:
        # --- __aexit__ 部分 ---
        print(f"关闭到 {host} 的连接")
        await asyncio.sleep(0.1)

async def test():
    async with get_connection("127.0.0.1") as conn:
        print(f"执行任务: {conn['host']}")
```

# 为什么必须用异步初始化/关闭？

在传统的 `with` (PEP 343) 中，`__enter__` 和 `__exit__` 是同步函数。如果你的资源关闭操作（如向服务器发送 `QUIT` 命令）涉及网络 IO，同步阻塞会导致整个事件循环挂起。

|**特性**|**with (同步)**|**async with (异步)**|
|---|---|---|
|**方法**|`__enter__` / `__exit__`|`__aenter__` / `__aexit__`|
|**调用方式**|阻塞式调用|挂起式调用 (`await`)|
|**适用场景**|本地文件、内存锁|网络请求、数据库、异步驱动|

# 常见应用场景

- **HTTP 请求**: `aiohttp.ClientSession()` 必须配合 `async with` 使用，以确保连接池在请求结束后正确回收。
    
- **异步锁**: `asyncio.Lock()` 使用 `async with` 自动获取和释放锁，防止死锁。
    
- **数据库引擎**: 如 `motor` (MongoDB) 或 `asyncpg` (PostgreSQL) 的连接管理。
    

> **小贴士**：如果你在 `async with` 内部忘记写 `await` 某些操作，或者尝试对不支持异步协议的对象使用 `async with`，Python 会抛出 `AttributeError: __aenter__`。
