实现一个异步资源管理器（Asynchronous Context Manager）需要遵循 Python 的 `async with` 协议。这要求类中必须实现两个特殊方法：`__aenter__` 和 `__aexit__`。

在异步编程中，这种模式非常强大，因为它能确保即使在发生异常的情况下，资源（如文件、数据库连接或网络套接字）也能被正确关闭。

#  异步上下文管理器协议

- **`__aenter__(self)`**: 在进入 `async with` 代码块时被调用。它通常负责异步打开资源，并返回资源实例。
    
- **`__aexit__(self, exc_type, exc_val, exc_tb)`**: 在退出代码块时被调用。它负责清理工作。如果代码块内抛出了异常，异常信息会通过参数传递进来。
    
# 异步文件读写工具

由于 Python 原生的 `open()` 是阻塞的，真正的异步文件 I/O 通常需要像 `aiofiles` 这样的库。但为了演示协议原理，我们可以用 `asyncio.to_thread` 来模拟在一个线程池中进行异步操作。

```python
import asyncio
import os

class AsyncFileWriter:
    def __init__(self, filename, mode='w'):
        self.filename = filename
        self.mode = mode
        self.file = None

    # 进入时：打开文件
    async def __aenter__(self):
        print(f"--- 正在异步打开文件: {self.filename} ---")
        # 模拟 I/O 耗时并异步打开文件
        self.file = await asyncio.to_thread(open, self.filename, self.mode, encoding='utf-8')
        return self  # 返回值会被赋值给 'as' 后面的变量

    # 执行操作：写入内容
    async def write_data(self, content):
        if self.file:
            print(f"正在写入数据...")
            await asyncio.to_thread(self.file.write, content)

    # 退出时：关闭文件
    async def __aexit__(self, exc_type, exc_val, exc_tb):
        print(f"--- 正在清理并关闭文件 ---")
        if self.file:
            await asyncio.to_thread(self.file.close)
        
        # 处理异常逻辑
        if exc_type:
            print(f"检测到异常: {exc_val}，资源已安全关闭。")
        
        # 返回 True 会阻止异常向上抛出，通常保持默认（None/False）让外部感知错误
        return False

# 使用示例
async def main():
    try:
        async with AsyncFileWriter("demo.txt") as writer:
            await writer.write_data("你好，这是通过异步上下文管理器写入的内容。")
            # 模拟一个潜在的错误
            # raise ValueError("手动触发一个错误")
    except Exception as e:
        print(f"外部捕获到错误: {e}")

asyncio.run(main())
```

# 为什么不直接用 `try...finally`？

虽然 `try...finally` 也能达到同样的目的，但 `async with` 有以下优势：

1. **代码更优雅**：逻辑封装在类内部，调用者不需要关心关闭细节。
    
2. **强制性**：它形成了一个清晰的作用域，减少了忘记关闭资源导致的内存泄漏。
    
3. **可组合性**：你可以轻松地嵌套多个 `async with`（例如同时打开源文件和目标文件）。

# `contextlib` 的简化写法

如果你觉得写一个类太麻烦，Python 提供了 `contextlib.asynccontextmanager` 装饰器，可以用生成器的方式快速实现：

```python
from contextlib import asynccontextmanager

@asynccontextmanager
async def simple_async_file(name):
    # __aenter__ 部分
    f = await asyncio.to_thread(open, name, 'w')
    try:
        yield f  # 暂停在这里，把 f 交给 async with 块
    finally:
        # __aexit__ 部分
        await asyncio.to_thread(f.close)
```

这种写法在逻辑简单时非常受欢迎。
