# 同步 vs 异步 (The "What")

- **同步 (Synchronous)**：任务按顺序执行。必须等前一个任务彻底完成，后一个任务才能开始。就像在银行柜台排队，前一个人没办完，你就得等着。
    
- **异步 (Asynchronous)**：任务可以“并行”启动。不需要等待前一个任务结束即可开启下一个任务。就像在餐厅扫码点餐，大家可以同时下单，不需要等前一桌吃完。

```python
import asyncio
import time

# 模拟一个耗时的 I/O 任务
def sync_task(name):
    print(f"同步任务 {name} 开始...")
    time.sleep(1)  # 阻塞当前线程
    print(f"同步任务 {name} 完成")

async def async_task(name):
    print(f"异步任务 {name} 开始...")
    await asyncio.sleep(1)  # 非阻塞挂起
    print(f"异步任务 {name} 完成")

# --- 同步执行函数 ---
def run_sync():
    start = time.perf_counter()
    print("\n>>> 开始运行同步代码")
    for i in range(3):
        sync_task(i)
    end = time.perf_counter()
    print(f"同步总耗时: {end - start:.2f} 秒")

# --- 异步执行函数 ---
async def run_async():
    start = time.perf_counter()
    print("\n>>> 开始运行异步代码")
    # 同时启动 3 个任务
    tasks = [async_task(i) for i in range(3)]
    await asyncio.gather(*tasks)
    end = time.perf_counter()
    print(f"异步总耗时: {end - start:.2f} 秒")

if __name__ == "__main__":
    # 1. 运行同步实验
    run_sync()
    
    # 2. 运行异步实验
    asyncio.run(run_async())
```

# 阻塞 vs 非阻塞 (The "How")

- **阻塞 (Blocking)**：程序在等待某个操作（如网络响应）时，CPU 被挂起，什么也干不了。
    
- **非阻塞 (Non-blocking)**：程序在等待时，可以立即去处理其他逻辑，直到操作完成收到通知。



# 事件循环 (Event Loop)

这是 `asyncio` 的“心脏”。它是一个持续运行的循环，负责监听任务状态。

1. 它把所有任务（Coroutines）放入队列。
    
2. 当某个任务遇到 `await`（如等待 IO）时，事件循环会将其挂起，切换到队列中的下一个任务。
    
3. 当 IO 完成，事件循环会收到通知，并在合适的时机切回原任务继续执行。
