## 🛠 第一阶段：命令行生存 (第 1 - 7 天)

**核心目标**：彻底丢掉鼠标，能在文件系统中“盲操”。

|**天数**|**学习内容**|**核心指令**|
|---|---|---|
|**D1-D2**|**路径英雄**：理解根目录 `/`、家目录 `~` 与相对路径。|`pwd`, `cd`, `ls -lah`, `tree`|
|**D3-D4**|**文件管理**：创建、移动、重命名及彻底删除。|`mkdir -p`, `touch`, `cp -r`, `mv`, `rm -rf`|
|**D5**|**内容速览**：不打开文件也能看数据内容。|`cat`, `head -n 20`, `tail -f` (监控日志必备)|
|**D6-D7**|**搜索大法**：在几千个文件中瞬间定位。|`find . -name "*.py"`, `locate`|

> **💡 工程师技巧**：在 Obsidian 中建立一个 `Linux_Commands.md`，用表格记录下那些容易忘的参数（如 `ls` 的 `-h` 是人眼可读格式）。

## ⚙️ 第二阶段：系统掌控与权限 (第 8 - 15 天)

**核心目标**：理解 Linux 的安全逻辑，配置你的开发环境。

- **D8-D10：权限与身份**
    
    - 学习 `chmod 755` 和 `chown` 的含义。
        
    - 理解为什么永远不要以 `root` 身份运行日常程序。
        
- **D11-D12：软件包管理**
    
    - 熟练使用 `apt update` 和 `apt install`。
        
    - 尝试在 Ubuntu 上安装 **Python 虚拟环境** (`venv`)。
        
- **D13-D15：文本编辑神器 Vim**
    
    - **生存挑战**：学会如何在终端里进入文件、修改一行代码并保存退出 (`:wq`)。
        
    - 这对于在远程服务器上紧急修改 Flask 配置文件至关重要。

## 🐍 第三阶段：数据处理与 Python 环境 (第 16 - 23 天)

**核心目标**：让 Ubuntu 成为你量化分析和爬虫任务的最佳拍档。

- **D16-D18：管道符与流处理 (核心)**
    
    - 掌握管道 `|`：将一个命令的输出传给下一个（如：`ls | grep ".csv"`）。
        
    - 学习 `grep` 的正则搜索，这在分析爬虫产生的巨大日志文件时非常高效。
        
- **D19-D21：环境自动化**
    
    - 配置 `.bashrc` 或 `.zshrc`。为常用的长命令设置别名 (`alias`)。
        
    - 例如：`alias run_backtest='python3 main.py --mode backtest'`。
        
- **D22-D23：SSH 远程连接**
    
    - 学习 `ssh-keygen` 生成密钥。尝试从 Windows 使用密钥无密码登录 Ubuntu。

## 🚀 第四阶段：部署、监控与自动化 (第 24 - 30 天)

**核心目标**：实现“无人值守”的金融数据抓取与交易系统。

- **D24-D25：任务定时化 (Crontab)**
    
    - 编写一个简单的 Cron 任务，让你的 Python 脚本每天早上 9:00 自动抓取开盘数据。
        
- **D26-D27：系统监控**
    
    - 使用 `htop` 查看 CPU 占用，使用 `df -h` 监控硬盘（防止爬虫数据撑爆磁盘）。
        
    - 学习 `kill -9` 强制结束卡死的进程。
        
- **D28-D29：Shell 脚本初步**
    
    - 编写一个 `.sh` 脚本，一次性完成“更新系统 -> 启动爬虫 -> 备份数据”的流程。
        
- **D30：实战测试**
    
    - 在 Ubuntu 上部署一个极简的 Flask 应用，并尝试在 Windows 浏览器访问它。
